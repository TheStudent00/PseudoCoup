# Phase 2: The Full QuickFox Fuzzer (Unparsing)

## Overview
This phase handles the generation of exhaustive, compilation-ready target source files (the "Mega Probe") using `tree-sitter`'s absolute syntax blueprint. This eliminates the need to manually write semantic probes and guarantees that every capability of the target language is captured.

## Objectives
- Build an Unparser (Syntax Fuzzer) that reads `tree-sitter` definitions.
- Automatically generate a target language file where every `tree-sitter` node maps to a dedicated, syntactically valid function.

## Sub-Tasks

### 2a: Tree-Sitter Grammar Ingestion
- **Task:** Build a parser that reads a target language's `tree-sitter` definitions (e.g., parsing the `node-types.json` generated by `tree-sitter-go`).
- **Details:** The parser must extract every structural node and its required child fields.

### 2b: Target Code Unparser Engine
- **Task:** Implement the Fuzzer that translates a parsed `tree-sitter` node definition into syntactically valid target code.
- **Details:** For example, given the node definition for `binary_expression` with fields `left`, `operator`, `right`, generate a wrapping function named `Probe_BinaryExpression_Add` that returns `a + b`. 
- **Type Variations:** Generate multiple variants of the same syntax node (e.g., using `int`, `float64`, `string`) to capture potential overloaded execution behaviors built into the target compiler.

### 2c: Output the Mega Probe
- **Task:** Serialize all generated functions into a single, valid target file (e.g., `quickfox.go`).
- **Details:** Ensure the file compiles cleanly without syntax or type errors. This file will serve as the input for Phase 3.
